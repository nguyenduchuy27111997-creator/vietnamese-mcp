Create `packages/shared/package.json`:
```json
{
  "name": "@vn-mcp/shared",
  "version": "0.0.1",
  "type": "module",
  "exports": {
    ".": "./src/index.ts",
    "./errors": "./src/errors/index.ts",
    "./http-client": "./src/http-client/index.ts",
    "./mock-engine": "./src/mock-engine/index.ts",
    "./test-helpers": "./src/test-helpers/index.ts"
  },
  "scripts": {
    "build": "tsdown src/index.ts --format esm --dts",
    "dev": "tsx watch src/index.ts"
  },
  "engines": { "node": ">=20" }
}
```

Note: The `exports` map uses source `.ts` paths for workspace development (TypeScript project references resolve these). The `build` script produces bundled output for publishing later. No `./types` subpath — per CONTEXT.md, each server owns its own Zod schemas.

Create `packages/shared/tsconfig.json`:
```json
{
  "extends": "../../tsconfig.base.json",
  "compilerOptions": {
    "outDir": "./build",
    "rootDir": "./src",
    "composite": true
  },
  "include": ["src/**/*"]
}
```

Create `eslint.config.js` using ESLint flat config format:
```javascript
import js from '@eslint/js';
import tseslint from 'typescript-eslint';

export default [
  js.configs.recommended,
  ...tseslint.configs.recommended,
  {
    rules: {
      'no-console': ['error', { allow: ['error', 'warn'] }],
    },
  },
  {
    ignores: ['build/**', 'dist/**', 'node_modules/**', '*.config.*'],
  },
];
```

The `no-console` rule with `allow: ['error', 'warn']` means:
- `console.log()` = lint ERROR (fails CI)
- `console.error()` = allowed (MCP stderr is safe)
- `console.warn()` = allowed
