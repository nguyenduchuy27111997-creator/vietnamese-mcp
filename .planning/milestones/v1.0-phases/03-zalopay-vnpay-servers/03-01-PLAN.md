**signatures.ts** — Import `signHmacSha256` from `@vn-mcp/shared`. Create 4 functions with hardcoded pipe-delimited field ordering:

1. `buildCreateSignature(params, key1)` — Fields: `app_id|app_trans_id|app_user|amount|app_time|embed_data|item` (7 fields). Returns `signHmacSha256(key1, raw)`.

**tools/validateCallback.ts** — Tool name: `zalopay_validate_callback`. Description: `"Validate a ZaloPay callback payload by verifying its HMAC-SHA256 MAC using key2. Returns validation result with parsed transaction details."`. Zod schema:
- `callbackData`: `z.string().min(1).describe('Raw ZaloPay callback JSON body as a string')`

Handler logic (mirrors MoMo's `validateIpn.ts` pattern):
1. Try `JSON.parse(args.callbackData)` — if fails, return `formatToolError(new Error('Invalid JSON in callbackData'))`
2. Extract `mac` field from parsed object and `data` field (which is a JSON string)
3. Get credentials via `getZaloPayCredentials()`
4. Compute expected MAC: `buildCallbackSignature(parsed.data, credentials.key2)` — note: ZaloPay callback MAC signs the raw `data` string with key2
5. If `expectedMac === mac`: return `{ valid: true, ...JSON.parse(parsed.data) }` (parse the data string to extract transaction fields)
6. If mismatch: return `{ valid: false, reason: 'MAC mismatch' }`

Import `getZaloPayCredentials` from `../credentials.js` and `buildCallbackSignature` from `../signatures.js`.
