**createPayment.ts:**
Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`, `z` from `zod`, `formatToolError` from `@vn-mcp/shared`, `momoClient` from `../client.js`.

Export `function register(server: McpServer)` that calls `server.tool()` with:
- name: `'momo_create_payment'`
- description: `'Create a MoMo payment link (QR code, wallet, or ATM). Returns payUrl for the customer to complete payment.'`
- schema (inline Zod, per INFRA-05):
  - `amount`: `z.number().int().positive().describe('Payment amount in VND (integer, e.g. 150000)')`
  - `orderInfo`: `z.string().min(1).describe('Order description shown to the customer')`
  - `redirectUrl`: `z.string().url().optional().describe('URL to redirect after payment')`
  - `ipnUrl`: `z.string().url().optional().describe('Server callback URL for payment notification')`
  - `requestType`: `z.enum(['captureWallet', 'payWithATM', 'payWithCC']).optional().describe('Payment method (default: captureWallet for QR code)')`
  - `extraData`: `z.string().optional().describe('Additional data to pass through')`
- handler: `async (args) => { try { const result = await momoClient.createPayment(args); return { content: [{ type: 'text' as const, text: JSON.stringify(result) }] }; } catch (err) { return formatToolError(err); } }`

**queryStatus.ts:**
Same pattern. Export `register(server)`:
- name: `'momo_query_status'`
- description: `'Check the status of a MoMo payment transaction by orderId.'`
- schema: `orderId`: `z.string().min(1).describe('The orderId from momo_create_payment response')`
- handler: delegates to `momoClient.queryStatus(args)`, wraps in text content, catches errors with `formatToolError`.

**refund.ts:**
Same pattern. Export `register(server)`:
- name: `'momo_refund'`
- description: `'Refund a MoMo transaction (full or partial) by transId.'`
- schema:
  - `transId`: `z.number().int().positive().describe('Transaction ID from the original payment (numeric, e.g. 2350000001)')`
  - `amount`: `z.number().int().positive().describe('Refund amount in VND (must be <= original amount)')`
  - `description`: `z.string().optional().describe('Reason for the refund')`
- handler: delegates to `momoClient.refund(args)`, wraps in text content, catches errors.

**validateIpn.ts:**
This tool is SPECIAL â€” it does real HMAC verification, not mock delegation. Import `getMomoCredentials` from `../credentials.js` and `buildIpnSignature` from `../signatures.js`.

Export `register(server)`:
- name: `'momo_validate_ipn'`
- description: `'Validate a MoMo IPN (Instant Payment Notification) payload by verifying its HMAC-SHA256 signature. Returns validation result with parsed transaction details.'`
- schema: `ipnBody`: `z.string().min(1).describe('Raw IPN JSON body as a string')`
- handler:
  1. Parse `ipnBody` as JSON. If parse fails, return `formatToolError(new Error('Invalid JSON in ipnBody'))`.
  2. Extract `signature` field from parsed payload.
  3. Get credentials via `getMomoCredentials()`.
  4. Build expected signature using `buildIpnSignature()` with the 13 payload fields and `credentials.secretKey`.
  5. Compare `expectedSignature === signature` (constant time not needed for mock â€” this is validation, not auth).
  6. If valid: return `{ valid: true, orderId, amount, transId, resultCode, message }` from the parsed payload.
  7. If invalid: return `{ valid: false, reason: 'Signature mismatch' }`.
  8. Wrap result in `{ content: [{ type: 'text', text: JSON.stringify(result) }] }`.
  9. Do NOT use `isMockMode` â€” IPN validation always uses real HMAC logic (per CONTEXT.md: "Real HMAC verification in mock mode").
