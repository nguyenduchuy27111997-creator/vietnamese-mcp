<objective>
Add server scoping to API keys at the gateway level. A new `allowed_servers` column on api_keys restricts which MCP servers a key can access. The auth middleware populates this into AuthContext (and KV cache). The MCP route handler checks scope and returns 403 for out-of-scope requests. Keys CRUD endpoints accept and return the field.

Purpose: SCOPE-01 (keys restricted to specific servers) and SCOPE-03 (gateway enforces scope with 403)
Output: Migration SQL, updated types, auth middleware, scope enforcement in index.ts, updated keys routes, scope tests
</objective>
