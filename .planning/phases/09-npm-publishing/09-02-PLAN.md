  <action>
    CRITICAL: Publish @vn-mcp/shared FIRST. Server packages depend on it.

    Step 1: Publish @vn-mcp/shared
    ```
    cd packages/shared
    npm publish --access public
    ```
    Wait for success. Then verify it exists on the registry:
    ```
    npm view @vn-mcp/shared version
    ```
    Must return `1.0.0`.

    Step 2: Publish all 5 server packages (order does not matter among them):
    ```
    cd servers/mcp-momo-vn && npm publish --access public
    cd servers/mcp-viettel-pay && npm publish --access public
    cd servers/mcp-vnpay && npm publish --access public
    cd servers/mcp-zalo-oa && npm publish --access public
    cd servers/mcp-zalopay-vn && npm publish --access public
    ```

    Step 3: Verify all 6 are published:
    ```
    npm view @vn-mcp/shared version
    npm view @vn-mcp/mcp-momo-vn version
    npm view @vn-mcp/mcp-viettel-pay version
    npm view @vn-mcp/mcp-vnpay version
    npm view @vn-mcp/mcp-zalo-oa version
    npm view @vn-mcp/mcp-zalopay-vn version
    ```
    All must return `1.0.0`.

    Step 4: Smoke test standalone install outside the monorepo:
    ```
    mkdir /tmp/vn-mcp-smoke-test && cd /tmp/vn-mcp-smoke-test
    npm init -y
    npm install @vn-mcp/mcp-momo-vn
    ```
    Verify install succeeds with exit code 0. Check that @vn-mcp/shared was pulled from registry:
    ```
    ls node_modules/@vn-mcp/shared/build/index.js
    ls node_modules/@vn-mcp/mcp-momo-vn/build/index.js
    ```

    Step 5: Test that the server binary runs:
    ```
    cd /tmp/vn-mcp-smoke-test
    MOMO_SANDBOX=true npx mcp-momo-vn &
    sleep 2
    kill %1 2>/dev/null
    ```
    The server should start without import errors. It will hang on stdio waiting for input — that is correct behavior. Kill it after confirming no crash.

    Step 6: Clean up:
    ```
    rm -rf /tmp/vn-mcp-smoke-test
    ```
  </action>
