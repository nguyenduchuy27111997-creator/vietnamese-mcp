  <action>
    Create `apps/gateway/src/routes/usageExport.ts`:

    1. Create a Hono router `usageExportRouter` typed with GatewayEnv
    2. GET `/` handler:
       a. Extract `start` and `end` query params (YYYY-MM-DD format)
       b. Validate both are present, valid dates, start <= end, range <= 366 days. Return 400 JSON error if invalid.
       c. Get userId from `c.get('auth').userId`
       d. Look up all active (non-revoked) api_key ids for the user via Supabase (same pattern as usage.ts)
       e. If no keys, return empty CSV with just the header row
       f. Query Tinybird SQL endpoint (`${host}/v0/sql`) with this query:
          ```sql
          SELECT
            toDate(timestamp) AS date,
            server,
            tool,
            count() AS call_count
          FROM tool_calls
          WHERE api_key_id IN ({comma-separated quoted key ids})
            AND timestamp >= '{start} 00:00:00'
            AND timestamp < '{end + 1 day} 00:00:00'
          GROUP BY date, server, tool
          ORDER BY date, server, tool
          FORMAT CSVWithNames
          ```
          Use `Authorization: Bearer ${c.env.TINYBIRD_TOKEN}` header.
          Use `c.env.TINYBIRD_HOST ?? 'https://api.tinybird.co'` for the host.
       g. If Tinybird returns ok, stream its response body directly (it already returns CSVWithNames format which has the header). But rename columns if needed — Tinybird CSVWithNames uses the column aliases, so "date","server","tool","call_count" should match.
       h. If Tinybird fails or is unreachable, fall back to generating a simple CSV from KV monthly totals: one row per key per server with the monthly count distributed evenly across the date range. This fallback ensures the endpoint works even without Tinybird configured.
       i. Set response headers:
          - `Content-Type: text/csv`
          - `Content-Disposition: attachment; filename="usage-export-${end}.csv"`
          - `Cache-Control: no-store`

    3. For the test file, mock `getServiceRoleClient` and `fetch` (for Tinybird). Test the validation logic and CSV output format. Use the existing test patterns from `apps/gateway/src/__tests__/usage-route.test.ts` as reference for mocking Hono context.

    IMPORTANT: Sanitize key IDs in the SQL query to prevent injection — only allow UUIDs (alphanumeric + hyphens). If a key ID doesn't match UUID format, skip it.
