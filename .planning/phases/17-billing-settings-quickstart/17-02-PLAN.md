Replace SettingsPage.tsx placeholder with a full settings page. Follow the OverviewPage layout pattern.

**Page structure:**

1. **Header** — "Settings" h1 with Settings icon and "Manage your account and preferences." subtitle.

2. **Profile Section** — Card with:
   - CardTitle "Profile"
   - **Email row**: Label "Email" + the user's email displayed in a read-only Input (or just a `<p>` with `text-sm font-medium`). Get email from `supabase.auth.getSession()` in a useEffect on mount, stored in `useState<string>('')`.
   - **Password change form**:
     - Three Input fields: "New Password" (`type="password"`, min 6 chars), "Confirm Password" (`type="password"`)
     - Note: Supabase `updateUser` does NOT require current password when user has active session, so only need new + confirm fields
     - Submit Button "Update Password"
     - State: `newPassword`, `confirmPassword`, `passwordLoading`, `passwordMessage` (success/error string), `passwordError` (boolean for styling)
     - On submit: validate newPassword.length >= 8, validate newPassword === confirmPassword, then call `supabase.auth.updateUser({ password: newPassword })`. On success set message "Password updated successfully" (green). On error set message from error.message (red). Clear form on success.
     - Display message below button: `<p className={passwordError ? 'text-destructive' : 'text-green-500'} ...>`

3. **Danger Zone** — Card with red border styling:
   - `className="border-destructive/50"` on the Card
   - CardTitle "Danger Zone" with `text-destructive` class
   - CardDescription: "Permanently delete your account and all associated data."
   - AlertDialog wrapping a "Delete Account" Button (`variant="destructive"`):
     - AlertDialogTrigger: the delete button
     - AlertDialogContent with:
       - AlertDialogHeader: title "Are you sure?" description "This action cannot be undone. This will permanently delete your account, API keys, and usage data."
       - A text input asking user to type "DELETE" to confirm. State: `deleteConfirm`
       - AlertDialogFooter: AlertDialogCancel "Cancel" + AlertDialogAction (styled destructive) "Delete Account" — DISABLED unless `deleteConfirm === 'DELETE'`
       - On confirm: call `supabase.auth.signOut()` (soft delete — just sign out and let them know to contact support for full deletion). Set a brief loading state.
       - Note: For the AlertDialogAction with conditional disable, use `onClick` with `e.preventDefault()` when disabled, or use a regular Button inside the footer instead of AlertDialogAction to allow the disabled prop.

**Implementation detail for AlertDialog action disable:**
Since AlertDialogAction auto-closes, use a regular `<Button>` with `variant="destructive"` inside AlertDialogFooter instead of AlertDialogAction. This lets you control the disabled state and onClick independently. Wrap close behavior manually: after sign-out succeeds, the auth state change will redirect to login.

**Imports:** Settings from lucide-react. Card/CardHeader/CardTitle/CardDescription/CardContent, Button, Input from shadcn/ui. AlertDialog and all sub-components from alert-dialog. supabase from supabase.ts. useState, useEffect from react.

**Layout:** `max-w-2xl mx-auto space-y-6` for narrower settings layout (matches original placeholder width pattern).
